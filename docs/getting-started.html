Chapter 1. Summary

Algorithm Elf™ is designed to help the students understanding the algorithm in the Data Structure and provide more materials used by the teachers to design their course, for examples, the anamate process of the algorighm implemented.

Main features:

  • Algorithm Animation, as well as the view of stack changing and source code.

  • Using datapool to config the initial parameters of each algorithm.

  • The common data structure: Queue, Tree, and Graph etc. can be shown in graphic mode.

  • You can add new algorithm written by pascal, build it and play it. And you can debug the algorithm, set the breakpoints and watchpoints. Algorithm Elf can displays the data changes in the implementation process, using an intuitive way of showing the problem of algorithm.

Chapter 3. Using the examples

Some algorithms, for examples, hanoi, travel binary tree, quick sort, merge sort ect. have been implemented by the Algorithm Elf™. You can use one of them to learn the usage of Algorithm Elf.

Animation Process of Algorithm

  1. Click File → Open Examples → hanoi.

  2. Click toolbar buttonPlay, the player window shows.

  3. In the player window, click buttonStart, the algorithm Hanoi is ready for running.

  4. Click buttonRun, running the algorithm.

  5. The graphic data view will show the process of implementation.

Tip

You can set the interval of each action by typing the micro-seconds in the Control Button Delay.

Step the algorithm

When the algorithm is ready, not clicking button Run, but Step, it will enter the Step Mode.

  • Click button Step once, step one line. The graphic data view will show the data in the current context.

  • In the Step Mode, click buttonRun, it will enter the Animation Mode.

  • In the Animation Mode, click button Pause, it will switch the Step Mode.

Change the arguments of algorithm

You can set the alogrithm arguments by Datapool in Algorithm Elf™. For examples, we want to use 5 plates to show the haoni algorithm,

  1. Click the labelParameters in the right screen, it will list all the arguments used by the hanoi: n, x, y, z

  2. Select the parameter n, click the value 3, then type new value: 5

  3. Select the parameterx, in the property list, set the Size to 5

  4. So the parameter y and z does.

  5. Click button Apply in the bottom.

  6. Click button Start, play the algorithm again.

Chapter 4. Writing your algorithm

You can write your algorithm in the Algorithm Elf™ by Pascal Language.

Note

Not all the standard pascal elements can be used in your algorithm

New an algorithm

  1. Run Algorithm Elf™ at first.

  2. In the main window, click button New.

  3. Type the following text in the edit control: 

    
program locate_sequence_list;
type
  sqlist: array [ 1..3 ] of integer;
var
  va: sqlist;
  k: integer;
  index: integer;

  function loc_sqlist(var va: sqlist; x : integer) : integer;
    var
      i : integer;
    begin
      i := 1;
      while i <= 3 and va[i] <> x do
        i := i + 1;
      if i > 3 then
        result := 0
      else
        result := i
    end;

begin
  va[1] := 5;
  va[2] := 8;
  va[3] := 9;  
  k := 8;
  index := loc_sqlist(va, k);
end.

  4. Click button Save, save the algorithm.

Compile Algorithm

Click button Compile to build the algorithm. If something is wrong, the error message will be shown in the right area. Fix your algorithm until it can be built successfully.

Play Algorithm

After compiling pass, you can play the algorithm. What you need to do is to click the button Play.

About the pascal language supported by Algorithm Elf, see also the section Algorithm Language.

Chapter 5. Using Datapool

If you want to config the algorithm arguments when you play the algorithm, you need use Datapool.

When you delcare one variable in the algorithm, a prefix '@' of the var name leads to initialize the variable value to the value from the same name item in the Datapool.

  1. Open the algorithm we wrote just now.

  2. Modify the algorithm as the following: 

    
program locate_sequence_list;
type
  sqlist: array of integer;
var
  @va: sqlist;
  @k: integer;
  index: integer;

  function loc_sqlist(var va: sqlist; x : integer) : integer;
    var
      i : integer;
    begin
      i := Low(va);
      while i <= High(va) and va[i] <> x do
        i := i + 1;
      if i > High(va) then
        result := 0
      else
        result := i
    end;

begin
  index := loc_sqlist(va, k);
end.

  3. Save the algorithm.

  4. Compile the algorithm.

  5. Click button Play, enter the player window.

  6. In the player window, click toolbar button Parameters, enter the parameter dialog. It will show two parameters: va, k

  7. Select the parameter va

  8. Set the property Type to Array

  9. Set the property Size to 5

  10. Click the button Random Data, it will generate the random data to initalize this array.

  11. Select the parameter k

  12. Click the graphic node, type the new value: 15

  13. Click button Apply.

  14. Now you can play the algorithm using the new parameters.

Chapter 6. Breakpoint and Watchpoint

Sometimes you need set breakpoints or watchpoints to find the data error in the algorithm.

Watchpoint means if the expression is changed, the execution will stop.

Only when running the algorithm, you cat set the breakpoints. The following actions happen in the player window.

Add breakpoints

Click one of the line in the source code window, type b, or click the left-most line, it will inisert a breakpoint

Click toolbar button Breakpoint to show the breakpoint list

Add Watchpoint

In the source code window, select any of expression, then type w or in the breakpoint view, type the expresstion in the botton of breakpint view, and click Watch

Remove breakpoint or watchpoint

Click toolbar button Breakpoint, it will show all the breakpoints and watchpoints.

Click the line to select the breakpoint or watchpoint, it will be marked as '*' in the beginning of the line.

Clic button Remove, remove the selected item.

A.1. Algorithm File Structure

program NAME;

const

  CONSTANT DEFINITION;         { optional }

type

  TYPE DEFINITION;             { optional }

var

  DECLARATION;                 { optional }

  FUNCTIONS OR PROCEDURES;     { this section must be located here }

begin             

  MAIN PROCEDURE

end.

A.2. Data Type

Simple Data Type

Integer, Real, Char, Boolean, String

For examples, 

var
  r: Real;
  i: Integer;
  c: Char;
  b: Boolean;

Complex Data Type

Set, Enum, Record, Array

For examples, 

var
  set1: set of Integer;
  arr1: array of Integer;       { Dynamic array }
  arr2: array [0..10] of String;
  r1: record of
       name: String;
       age: integer;
     end;
  e: (apple, pear, banana, orange, lemon);

User defined data type and pointer 

type
  x = Integer;
  y = x;
  ...

type 
  a = Array [1..10] of Integer;
  b = record
        x: Integer;
        y: Char
      end;

type 
  a = ^b;
  b = record
        x: Integer;
        y: Char;
        z: a
      end;
var
  pointertob: a;
  pInt : ^Integer;

new(pointertob);

pointertob^.x := 10;
pointertob^.y := 'A';
pointertob^.z := nil;

A.3. Control Statement and Expression

Control Statement. if, while, repeat, case, for 


    while a<>b do 
      writeln('Waiting');
 
    if a > b then
      writeln('Condition met')
    else 
      writeln('Condition not met');
 
    for i := 1 to 10 do
      writeln('Iteration: ', i:1);
 
    repeat
      a := a + 1
    until a = 10;
 
    case i of
      0: write('zero');
      1: write('one');
      2: write('two');
    end;

Expression.  


      Number Expression: +, -, *, / 

         1 + a, 2 * 4, 3.5 / 6

      Boolean Expression: and, or, not, <, >, <=, >=, <>
      
         a and b 
         a = b
      
      String Expression: +
      
         'abc' + 'efg'

      Set Operator: in 
         
          a in [1,2,3]  

      Pointer Operator: ^ , @
          
          p^ := s;
          p  := @i;
          p  := addr(x);

      Visit record: .

          s.name := 'jondy';

      Assignment: :=

          s := 2;

A.4. Function and Procedure

For examples, 

  function next(k: integer): integer;
  begin
    result := k + 1
  end;

  procedure print(var j: integer); 
  begin
    writeln('The total is: ', j);
    j := next(j)
  end;

Note

In the function, the return variable is keyword 'result', not function name.

Note

You can not define procedure or function in the function or procdeure body.

A.5. Extension: Datapool

Datapool is one of extension for the standard pascal. The code is simple: if one variable is start with '@', its intial value will get from the value of same name in the datapool. 

var
    @x: Integer;
begin
    writeln('Get the value from datapool: ', x);
end.
